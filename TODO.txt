Prompts:
	Book are animated to open in the inventory. If we don't use the correct animated mesh, the book fails to open. We need to use skyrim book mesh inventory model INAM, BUT we ideally need to apply the oblivion texture for the book front/back covers and spine.

	The biggest issue with navmesh generation are many holes that shouldn't be there and disconnected sections. Ideally the navmesh fills all those holes with good large triangles, while avoiding walls and obstacles. The issue is the current system is really bad at finding what is an unpassable wall and what is a floor, for example anvilfightersguild (temp/anvilfg_navmesh.png) holes are actually walls and correct, but the walls don't show on the image. In fact, I earlier painted the approximate wall location (nav_AFG_withpaintedwalls.png).
	We also need to be better at trusting the pathgrid height. If the pathgrid is going over a static bridge outdoors for example, we know it is walkable and that is the approximate height of the floor. It will also incorrectly attach second floor triangles with the first floor navmesh.
	Here is my plan: Add a script to better categorize the meshes and even parts of meshes as floors obstacles, walls, and things that are ok to path over. I'm sure there is a way to do this programmatically.
	Then once you have that, you'll have to load the actual landscape topography and mesh triangles to generate a good navmesh that properly avoids obstacles. This is a complex task, but I know you can do it!
	
	Skyrim bsa files have a file size limit (2,147,483,648 bytes). The BSA builder should respect that limit. If the limit will be reached create a dummy .esl file or more (oblivion_loader.esl, oblivion_loader_1.esl) to fit everything else into

	Move any external dependencies used by our pipeline to the external folder. bsarch.exe, LODGENx64.exe, papyrus.exe are the ones that come to mind but it's possible there are more. Ensure they are all documented properly in the readme license section.
