REMAINING buckets (unchanged root causes from before, still to fix):

	[MASTERFILE] 1944 total:
	1) (512x) "Could not find worldspace (X) in load for Location 'X' (X) in file Oblivion.esm."
		Generated LCTN records point at a worldspace FormID that isn't loaded/converted (0x0100003C Tamriel and gate-related worldspaces most common). See memory: Exterior cell names / LCTN worldspace+marker construction.
	2) (298x + 297x = 595x) "Armor/ArmorAddon 'X' (X) used invalid Biped Object slot 44 for race 'DefaultRace'."
		Slot-44 body-partition patch (see [FORMS] entry below) assigns biped slot 44 to items on 'DefaultRace', which doesn't define slot 44 in its body template — cosmetic-only CK complaint, but confirms the slot-44 patch is applied indiscriminately rather than only to races that support it.
	3) (245x) "Container object (X) has a count of less then 1 so no item is really added but will cause issues in game on owner object "X"."
		Converted container entries have Count <= 0 — an inventory-count field is being zeroed/lost during CONT/vendor-chest conversion.
	4) (139x) "Could not find FootstepSet (X) on ArmorAddon 'X' (X)."
		ArmorAddon FootstepSet FormID reference (Oblivion has no direct equivalent) isn't populated/resolved — expected gap, needs a default FootstepSet assignment per armor material in ARMA conversion.
	5) (135x + 135x = 270x) "Could not find male/female voice type (X) for race 'X' (X)."
		Generated races (per-NPC custom races like TES4SEUncleLeoRace) don't have VTYP FormIDs wired up for both genders — voice type routing gap for custom/unique-NPC races specifically (distinct from the main race-based voice routing already tracked in memory).
	6) (130x) "Unable to find Package Target Reference (00000000) on owner object "X"." — residual after the fix above; a smaller remaining set of packages still have an unresolved/zeroed target reference.
	7) (19x) "Unable to find Package Location Reference (X) on owner object "X"." — same root cause as #6 but for the package's location field instead of target field.
	8) (14x) "Unable to find Leveled Object Form (00000000) for owner object "X"." — leveled list entry with an unresolved/zeroed FormID, same shape as #6/#7.
	9) (~11x) "Reference attached to wrong cell for its location: 'X' (X) (x, y) in WorldSpace 'X' (X)" — REFR's persistent-location cell doesn't match its actual grid coordinates; matches [CELLS] bucket below, likely same root cause.
	Singletons (not worth a bucket): duplicate Navmesh map ID, mapmarker ref missing map marker data, 2 stray Function-Info-TESForm lines, 1 stray Enchanting-cast-type line (see FIXED above — not worth re-chasing at this volume).
