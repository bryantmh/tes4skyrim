CK Warnings Analysis (CK_WARNINGS, 12789 lines, from CKPE load of converted Oblivion.esm):
Genericized every message (FormIDs/names/numbers stripped) and grouped identical patterns. 6 top-level
CKPE categories, ~13 root-cause buckets. Counts are occurrence counts in the log.

	[MASTERFILE] 7763 total:
	4) (512x) "Could not find worldspace (X) in load for Location 'X' (X) in file Oblivion.esm."
		Generated LCTN records point at a worldspace FormID that isn't loaded/converted (0x0100003C Tamriel and gate-related worldspaces most common). See memory: Exterior cell names / LCTN worldspace+marker construction.
	5) (298x + 297x = 595x) "Armor/ArmorAddon 'X' (X) used invalid Biped Object slot 44 for race 'DefaultRace'."
		Slot-44 body-partition patch (see [FORMS] entry below) assigns biped slot 44 to items on 'DefaultRace', which doesn't define slot 44 in its body template — cosmetic-only CK complaint, but confirms the slot-44 patch is applied indiscriminately rather than only to races that support it.
	7) (245x) "Container object (X) has a count of less then 1 so no item is really added but will cause issues in game on owner object "X"."
		Converted container entries have Count <= 0 — an inventory-count field is being zeroed/lost during CONT/vendor-chest conversion.
	8) (139x) "Could not find FootstepSet (X) on ArmorAddon 'X' (X)."
		ArmorAddon FootstepSet FormID reference (Oblivion has no direct equivalent) isn't populated/resolved — expected gap, needs a default FootstepSet assignment per armor material in ARMA conversion.
	9) (135x + 135x = 270x) "Could not find male/female voice type (X) for race 'X' (X)."
		Generated races (per-NPC custom races like TES4SEUncleLeoRace) don't have VTYP FormIDs wired up for both genders — voice type routing gap for custom/unique-NPC races specifically (distinct from the main race-based voice routing already tracked in memory).
	10) (19x) "Unable to find Package Location Reference (X) on owner object "X"." — same root cause as #2 but for the package's location field instead of target field.
	11) (14x) "Unable to find Leveled Object Form (00000000) for owner object "X"." — leveled list entry with an unresolved/zeroed FormID, same shape as #2/#10.
	12) (~15x) "Reference attached to wrong cell for its location: 'X' (X) (x, y) in WorldSpace 'X' (X)" — REFR's persistent-location cell doesn't match its actual grid coordinates; matches [CELLS] bucket below, likely same root cause.
	13) (~30x, long tail) Duplicate/variant "Unable to find Function Info TESForm" lines with extra trailing quoted text — same bug as #1, just messages with embedded quotes in the INFO text confusing simple parsing; not a separate bug.
	Singletons (not worth a bucket): duplicate Navmesh map ID, mapmarker ref missing map marker data.

	[ANIMATION] 3231 total, 1 pattern:
	"Animation 'X' on race 'X' has no weaponSwing/weaponLeftSwing event" — every converted attack animation is missing weapon-swing sound/event annotations. Likely systemic: annotation/event track isn't carried over (or generated) during animation conversion. Matches TODO "Minor Issues #2: Weapons have an attack animation of unknown instead of default."

	[QUESTS] 835 total:
	1) (806x) "Orphaned topic X in quest TES4DialogueGeneric (X)" [+ a few dozen in other named quests] — INFO/topic records exist but aren't linked into any quest's topic list (TPIC alias). Overwhelmingly TES4DialogueGeneric, consistent with the generic-greeting/topic-per-quest issues already tracked (see memory: One bark topic per quest, Greeting→response TCLT).
	2) (5x) "Could not find forced ref (X) for Ref Alias 'X'." — quest alias's forced reference FormID doesn't resolve; small-count, likely a handful of quests with unconverted/unmapped forced-ref FormIDs.

	[MAGIC] 519 total:
	1) (369x) "Magic Item (X) 'X' is AIMED but has no Magic Effects with Projectiles assigned." — converted spell/enchantment keeps an Aimed delivery type but its effects lost their Projectile (PROJ) reference during MGEF/ENCH conversion.
	2) (150x) "Magic Item (X) 'X' has no effects defined." — spell/enchantment converted with zero effects; effects list is being dropped for a subset of SPEL/ENCH records.

	[EDITOR] 392 total, 1 pattern:
	"Editor ID 'X' for TYPE (X) is not unique, previous object (X) is type OTHERTYPE. Editor ID will be set to 'XDUPLICATE001'." — EditorID collisions between generated/converted records and existing Skyrim.esm records (LCTN 129x, BOOK 68x, SOUN 42x/29x, STAT 17x, INGR 14x, SPEL 12x, QUST 12x, etc.). Cosmetic (CK auto-renames), but signals the import isn't namespacing generated EditorIDs (e.g. LCTN/BOOK) distinctly enough from vanilla Skyrim names.

	[CELLS] 13 total, 1 pattern:
	"Ref is not in its persistence location 'X' (X)." — same underlying issue as MASTERFILE bucket #12 (REFR persistence/cell-location mismatch).

	[FORMS] 1 total:
	Truncated SNAM_ID chunk (>512 byte max) on the TES4 form — the auto-generated slot-44 body-split patch description in the CK's own patch-note field is too long and gets truncated. Harmless (log text only), but explains why bucket #5 above sees slot 44 on DefaultRace.
